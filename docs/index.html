              Index term generation
              

                Timehash recursively chooses the largest bucket that fits inside
                the range. Boundaries are refined with smaller buckets.
              

            

            

              Query term generation
              

                A query time creates one key per hierarchy level, from coarse
                buckets to exact minute-level buckets.
              

            

            

              Exact matching
              

                A document matches when any query key appears in the document's
                index keys, giving zero false positives and zero false negatives.
